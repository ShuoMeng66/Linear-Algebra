# Cherry Studio Agent 手动部署说明

这个文件对应的是 Cherry Studio 里 `智能体 -> 添加 Agent` 的手动创建界面，适合想把本项目做成“长期陪学型线代 Agent”的读者。

## 适合什么场景

- 想让 Agent 长期按本书主线陪学
- 想让它更像“会检查思路、会给提示、会补练习”的学习搭子
- 想按自己的模型服务商、权限模式和 Git Bash 路径来配置

## 建议填写

### 1. 名称

推荐填写：

```text
从零线代学习 Agent
```

如果你更想突出“补考冲刺”，也可以改成：

```text
从零线代补考陪学 Agent
```

### 2. 模型

优先选择你当前已经接好的、推理比较稳的模型。

如果你主要是：

- 日常伴学、知识点解释、基础例题：优先选速度快一点的模型
- 证明题拆步、综合题、反复追问：优先选推理更稳一点的模型

不管选哪一个，重点都不是“最贵”，而是“能稳定分步讲清楚”。

### 3. Git Bash

按你截图这类 Windows 环境，一般可用：

```text
D:\Git\Git\bin\bash.exe
```

如果本机路径不同，就点右侧 `选择` 按实际安装目录填写；如果界面能自动发现，直接用自动发现结果也可以。

### 4. 权限模式

如果这个 Agent 主要用来帮助读者学习，而不是操作本地项目，推荐先用：

```text
普通模式
```

如果你的 Cherry Studio 版本里还能选 `计划模式`，它也可以接受，因为会比全自动更可控。

不建议开启：

```text
全自动模式
```

原因很简单：这个 Agent 的主要用途是帮助读者学习线性代数，不是替你在本地环境里大范围执行命令。权限一旦开得太大，就会带来不必要的安全风险，比如误执行命令、误触文件操作，或者在你没细看的情况下做出不适合学习场景的动作。

所以这一项最稳的建议是：

- 优先 `普通模式`
- 如果你确实需要更强一点的过程引导，再考虑 `计划模式`
- 不要开启 `全自动模式`

### 5. 提示词

把下面这个文件里的全文复制进去：

```text
cherry-studio/linear-algebra-learning-agent.prompt.md
```

它已经专门针对这本书做了几件事：

- 保持口语化、陪学感，不会故意写得太专业
- 会优先按本书五章主线回答
- 会对参数分类、选择填空、正误判断先做一遍结果自检
- 会先保住学生做对的部分，再修第一处真正有问题的地方
- 会给补考同学更偏拿分的讲法和短练习

## 推荐使用方式

### 用法 1：例题精讲

```text
请按这本书第 3 章的风格讲这题，先说它在考什么，再一步一步算，不要跳步。
```

### 用法 2：只要提示

```text
我做到这里卡住了，先别给完整答案，只告诉我下一步先盯什么。
```

### 用法 3：思路批改

```text
这是我的解法，你先告诉我前面哪里是对的，再指出第一处真正有问题的地方。
```

### 用法 4：补考冲刺

```text
我在补考，请只按高频考点、常见丢分点和速判技巧给我讲。
```

## 两个很实用的小提醒

1. 如果你是拿它来陪着学，问题最好同时带上：
   - 章节名
   - 题目原文
   - 已做步骤
   - 卡住位置

2. 如果是选择题、填空题，最好直接补一句：

```text
请先说速判思路，再给答案。
```

这样它更容易按“学会”而不是“只报答案”的方式回答。
